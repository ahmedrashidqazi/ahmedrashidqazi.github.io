### **Design Directives: Retro-Futuristic Glitch Art**

**1. Overall Theme & Atmosphere:**
-   **Concept:** A dark-themed, retro-futuristic interface reminiscent of old CRT monitors, cyberpunk aesthetics, and digital glitch art. The feeling should be clean and modern, but with a distinct, "glitchy" personality.
-   **Visuals:** Think scanlines, subtle chromatic aberration, and sharp geometric lines combined with modern typography and layout.

**2. Color Palette:**
-   **Background:** A very dark charcoal, almost black (`#101010`).
-   **Primary Text:** A slightly off-white (`#EAEAEA`) for readability.
-   **Accent/Highlight Color:** A vibrant, neon cyan (`#00FFFF`). Use this for links, buttons, active navigation items, and key headings.
-   **Secondary Accent:** A neon magenta (`#FF00FF`). Use this for secondary highlights or hover effects to create a dynamic glitch effect.
-   **Borders/Lines:** A semi-transparent white (`rgba(255, 255, 255, 0.2)`).

**3. Typography:**
-   **Headings (`h1`, `h2`, `h3`):**
    -   Font Family: `'Fira Code', 'Courier New', monospace`.
    -   Weight: `500`.
    -   Letter Spacing: `1.5px`.
    -   Transform: `uppercase`.
-   **Body Text (`p`, `li`, etc.):**
    -   Font Family: `'Inter', 'Roboto', sans-serif`.
    -   Weight: `300` for standard text, `500` for bold/strong.
    -   Line Height: `1.6`.
-   **Navigation/Links:**
    -   Font Family: `'Fira Code', monospace`.
    -   Weight: `400`.

**4. Layout & Structure:**
-   **Asymmetrical Grid:** Reorganize the page sections into a more dynamic, asymmetrical layout. Avoid the standard single-column, centered approach. For example, the profile image could be on the left, with the name/title overlapping it slightly and the "About" section to the right.
-   **Negative Space:** Use generous negative space to let elements breathe and create a clean, uncluttered feel despite the "glitchy" effects.
-   **Visual Hierarchy:** Establish a clear visual hierarchy using typography, color, and positioning. The user's name/title should be the most prominent element on load.

**5. Micro-interactions & Animations:**
-   **Global Scanline Overlay:** Implement a faint, animated scanline effect over the entire page using a pseudo-element (`::after`) on the `body` with a repeating linear gradient and a subtle animation.
-   **Heading Glitch on Hover:** When a user hovers over a major heading (`h1`, `h2`), apply a "glitch" effect. This can be achieved with CSS using pseudo-elements (`::before`, `::after`) that rapidly shift position with different colors (the neon cyan and magenta).
-   **Scroll-Driven Animations (Intersection Observer):** As the user scrolls, fade in and slightly slide up each section (`<section>`) as it enters the viewport. Use the Intersection Observer API for performance.
-   **Button/Link Hover Effect:** On hover, links and buttons should have their text color change to the background color (`#101010`) and their background fill with the neon cyan (`#00FFFF`). Add a subtle `transform: translate(2px, 2px);` to give it a slight "jolt".
-   **Project Card Interactions:**
    -   On hover, the project card's border should glow with the neon cyan color.
    -   Apply a subtle `transform: perspective(1000px) rotateY(-5deg);` to give the card a slight 3D tilt on hover.
    -   The project description text, initially hidden or semi-transparent, should fade in clearly on hover.
-   **Timeline/Experience Section:** The connecting line in the timeline should "draw" itself in as the user scrolls down the section.

**6. Component-Specific Styling:**
-   **Navigation:** Keep it fixed to the top but make the background semi-transparent with a blur effect (`backdrop-filter: blur(10px);`) to see content scrolling underneath. The active link should be highlighted in the neon cyan.
-   **Profile Image:** Apply a duotone or a CRT-like filter effect to the profile image using CSS filters (`filter: grayscale(1) contrast(1.2) brightness(0.8);`) and blend a semi-transparent cyan layer on top using a pseudo-element.
-   **Research Modal:** Style the modal to match the new dark theme. Use sharp corners and the defined color palette. Add a blurred background for the content behind the modal.
-   **Snake Game Integration:** The Snake Game has a retro aesthetic that fits perfectly. Do not change its core JavaScript. Instead, style the *container* of the game. Give it a distinct border that looks like an old monitor bezel, perhaps with a subtle glowing border in the accent color. When the game is active, you could even intensify the global scanline effect.
